# TODOアプリケーション 戦略的設計

## 1. プロジェクトの目的

シンプルなTODOアプリケーションの開発を通じて、ドメイン駆動設計（DDD）の理解を深める。

## 2. アプリケーションの主な機能要件

*   **タスク管理:**
    *   タスクの作成、閲覧、編集、削除
    *   タスク属性: タイトル、説明、期日、ステータス（未着手、進行中、完了など）
*   **リスト管理:**
    *   リストの作成、閲覧、編集、削除
    *   タスクをリストに分類して管理
*   **ユーザー:**
    *   個人利用を想定

## 3. ユビキタス言語

*   **タスク (Task):** 実行すべき作業や事柄。
    *   **タイトル (Title):** タスクの名称。
    *   **説明 (Description):** タスクの詳細な内容。
    *   **期日 (DueDate):** タスクの完了目標日。
    *   **ステータス (Status):** タスクの進捗状況。
*   **リスト (List):** 複数のタスクを分類・管理するためのグループ。
*   **ユーザー (User):** アプリケーションの利用者。

## 4. ドメインとコンテキストマップ

*   **コアコンテキスト (Core Context):**
    *   **タスク・リスト管理 (Task & List Management):** タスクの作成、編集、削除、ステータス変更、期日設定、およびリストの作成、編集、削除、タスクのリストへの割り当てなど、タスクとリストに関する一連の操作を責務とする。


## 5. 技術スタック（`memory-bank`より）

*   フロントエンド: Next.js (App Router)
*   データベース: PostgreSQL
