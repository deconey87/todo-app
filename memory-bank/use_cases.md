# TODOアプリケーション ユースケース

**アクター:** ユーザー

## タスク関連ユースケース

1.  **UC001: 新しいタスクを作成する**
    *   目的: 新しいタスクをシステムに登録する。
    *   主要ステップ:
        1.  ユーザーがタスク作成を指示する。
        2.  システムがタスクリストの存在を確認する。
        3.  **タスクリストが存在しない場合 (最初のタスク作成時など):**
            1.  システムがタスク情報（タイトル、説明、期日など）と**新しいタスクリスト名**の入力フォームを表示する。
            2.  ユーザーがタスク情報と新しいタスクリスト名を入力し、登録を指示する。
            3.  システムが新しいタスクリストを作成する。
            4.  システムがタスクを保存し、作成されたタスクリストに関連付け、初期ステータスを「未完了」とする。
        4.  **タスクリストが既に存在する場合:**
            1.  ユーザーがタスクを追加したいタスクリストを選択する (またはシステムがデフォルトのリストを提示する)。
            2.  システムがタスク情報（タイトル、説明、期日など）の入力フォームを表示する。
            3.  ユーザーがタスク情報を入力し、登録を指示する。
            4.  システムがタスクを保存し、選択されたタスクリストに関連付け、初期ステータスを「未完了」とする。
2.  **UC002: タスク一覧を表示する**
    *   目的: 登録されているタスクの一覧を閲覧する。
    *   主要ステップ:
        1.  ユーザーがタスク一覧表示を指示する。
        2.  システムが登録済みのタスクを取得し、一覧形式で表示する。
3.  **UC003: タスク詳細を表示する**
    *   目的: 特定タスクの詳細情報を閲覧する。
    *   主要ステップ:
        1.  ユーザーが特定タスクの選択、または詳細表示を指示する。
        2.  システムが該当タスクの情報を取得し、表示する。
4.  **UC004: タスクを更新する**
    *   目的: 既存タスクの情報を変更する。
    *   主要ステップ:
        1.  ユーザーがタスク更新を指示する。
        2.  システムがタスク情報の編集フォームを表示する。
        3.  ユーザーが情報を編集し、更新を指示する。
        4.  システムがタスク情報を更新する。
5.  **UC005: タスクを削除する**
    *   目的: 既存タスクをシステムから削除する。
    *   主要ステップ:
        1.  ユーザーがタスク削除を指示する。
        2.  システムが確認を求め、ユーザーが承認する。
        3.  システムがタスクを削除する。
6.  **UC006: タスクを完了にする**
    *   目的: 未完了のタスクを完了状態に変更する。
    *   主要ステップ:
        1.  ユーザーがタスク完了を指示する。
        2.  システムがタスクのステータスを「完了」に変更する。
7.  **UC007: タスクを未完了に戻す**
    *   目的: 完了済みのタスクを未完了状態に戻す。
    *   主要ステップ:
        1.  ユーザーがタスクの未完了化を指示する。
        2.  システムがタスクのステータスを「未完了」に変更する。
8.  **UC008: タスクをステータスでフィルタリングする**
    *   目的: タスク一覧を特定のステータスで絞り込んで表示する。
    *   主要ステップ:
        1.  ユーザーがフィルタリング条件（ステータス）を選択する。
        2.  システムが条件に合致するタスクのみを一覧表示する。
9.  **UC009: タスクを期日でソートする**
    *   目的: タスク一覧を期日の昇順または降順で並び替える。
    *   主要ステップ:
        1.  ユーザーがソート条件（期日、昇順/降順）を選択する。
        2.  システムが条件に従ってタスクを並び替え、一覧表示する。

## タスクリスト関連ユースケース

10. **UC010: 新しいタスクリストを作成する**
    *   目的: 新しいタスクリストを作成する。
    *   主要ステップ:
        1.  ユーザーがタスクリスト作成を指示する。
        2.  システムがリスト名の入力フォームを表示する。
        3.  ユーザーがリスト名を入力し、登録を指示する。
        4.  システムがタスクリストを保存する。
11. **UC011: タスクリスト一覧を表示する**
    *   目的: 作成済みのタスクリストの一覧を閲覧する。
    *   主要ステップ:
        1.  ユーザーがタスクリスト一覧表示を指示する。
        2.  システムが登録済みのタスクリストを取得し、一覧形式で表示する。
12. **UC012: 特定のタスクリストにタスクを追加する**
    *   目的: 特定のタスクリストに新しいタスクを作成する。
    *   主要ステップ: (UC001に類似、リスト選択が加わる)
        1.  ユーザーがタスクを追加したいタスクリストを選択する。
        2.  ユーザーがタスク作成を指示する。
        3.  システムがタスク情報の入力フォームを表示する。
        4.  ユーザーが情報を入力し、登録を指示する。
        5.  システムがタスクを保存し、選択されたタスクリストに関連付ける。
13. **UC013: 特定のタスクリスト内のタスク一覧を表示する**
    *   目的: 特定のタスクリストに含まれるタスクの一覧を閲覧する。
    *   主要ステップ:
        1.  ユーザーが特定のタスクリストを選択する。
        2.  システムが該当タスクリスト内のタスクを取得し、一覧表示する。
14. **UC014: タスクリストの名前を変更する**
    *   目的: 既存のタスクリストの名前を変更する。
    *   主要ステップ:
        1.  ユーザーがタスクリスト名変更を指示する。
        2.  システムがリスト名編集フォームを表示する。
        3.  ユーザーが新しいリスト名を入力し、更新を指示する。
        4.  システムがタスクリスト名を更新する。
15. **UC015: タスクリストを削除する**
    *   目的: 既存のタスクリストを削除する（関連タスクも削除）。
    *   主要ステップ:
        1.  ユーザーがタスクリスト削除を指示する。
        2.  システムが確認を求め（関連タスクも削除される旨を明記）、ユーザーが承認する。
        3.  システムがタスクリストと関連するタスクを削除する。
16. **UC016: タスクを別のタスクリストに移動する**
    *   目的: 既存のタスクを、現在所属するタスクリストから別のタスクリストへ移動させる。
    *   主要ステップ:
        1.  ユーザーが移動したいタスクを選択する。
        2.  ユーザーがタスクの移動を指示する。
        3.  システムが移動先のタスクリストを選択肢として提示する。
        4.  ユーザーが移動先のタスクリストを選択する。
        5.  システムがタスクの所属タスクリスト情報を更新する。
